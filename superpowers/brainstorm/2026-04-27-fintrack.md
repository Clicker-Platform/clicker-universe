# Fintrack Module — Design Spec
> Brainstorming session: 2026-04-27
> Status: Approved, ready for implementation planning

---

## Keputusan Utama

| Keputusan | Pilihan |
|-----------|---------|
| Data scope | Per-site (bukan per-user), konsisten dengan semua modul lain |
| Database | Firestore (bukan Supabase) |
| Strategy | Fresh module, borrow AI layer pattern dari stocklens saja |
| AI scan | Default platform env key (`GEMINI_API_KEY`), bisa override per-site via Settings |
| Feature scope | Full feature set, implementasi per phase |

## Section 1 — Data Model (Firestore)

Semua path relatif terhadap `sites/{siteId}/`:

```
modules/fintrack/wallets/{id}
modules/fintrack/entries/{id}        ← bukan "transactions" untuk hindari naming clash
modules/fintrack/transfers/{id}
modules/fintrack/categories/{id}
modules/fintrack/budgets/{id}
modules/fintrack/goals/{id}
modules/fintrack/debts/{id}
modules/fintrack/recurring/{id}
modules/fintrack/private/config      ← API key config
```

### Catatan Penting
- **Saldo wallet tidak disimpan** — dihitung realtime via `getWalletBalance(siteId, walletId)` di `api.ts` yang sum entries dari Firestore
- **Categories**: default seed saat modul pertama diaktifkan + custom tambahan per-site
- **Transfers terpisah** dari entries agar tidak masuk kalkulasi cashflow
- **Tidak ada `user_id` per-row** — per-site, semua member bisnis lihat data yang sama

### Type Shapes (ringkasan)

```ts
// Wallet
{ id, nama, tipe, icon, warna, saldo_awal, createdAt }

// FintrackEntry
{ id, walletId, jumlah, jenis: 'pemasukan'|'pengeluaran', kategori,
  judul, catatan, buktiUrl, tanggal, isRecurring, recurringId, createdAt }

// Transfer
{ id, fromWalletId, toWalletId, jumlah, catatan, tanggal, createdAt }

// Category
{ id, nama, icon, warna, isDefault }

// Budget
{ id, categoryId, jumlah, periode: 'bulanan'|'tahunan', bulan, tahun, createdAt }

// Goal
{ id, nama, target, terkumpul, deadline, icon, createdAt }

// Debt
{ id, nama, jenis: 'hutang'|'piutang', jumlah, lunas, catatan, deadline, createdAt }

// Recurring
{ id, walletId, judul, jumlah, jenis: 'pemasukan'|'pengeluaran',
  kategori, frekuensi: 'harian'|'mingguan'|'bulanan'|'tahunan',
  tglMulai, tglBerikut, aktif, createdAt }
```

## Section 2 — File Structure

```
lib/modules/fintrack/
├── types.ts                      ← semua TypeScript types
├── constants.ts                  ← Firestore path constants + label maps
├── api.ts                        ← Firestore CRUD (client SDK) + getWalletBalance()
├── server/
│   └── receipt-scanner.ts       ← Gemini Vision scan struk (server-only, Firebase Admin)
└── admin/
    ├── DashboardPage.tsx         ← total saldo, cashflow, donut chart, bar chart
    ├── EntriesPage.tsx           ← riwayat entries, filter, search, pagination
    ├── WalletVaultPage.tsx       ← daftar dompet, saldo per wallet, transfer history
    ├── NewEntryPage.tsx          ← form manual + mode scan struk
    ├── AdvancedPage.tsx          ← 4 tab: Budget / Goals / Hutang & Piutang / Recurring
    └── SettingsPage.tsx          ← API key config

app/api/fintrack/
├── scan-struk/route.ts           ← POST multipart → receipt-scanner → return auto-fill data
└── settings/route.ts             ← GET/POST API key (Firebase Admin)
```

> Tidak ada `dashboard/route.ts` — aggregate dihitung di client via Firestore queries langsung.

## Section 4 — Phasing

### Phase 1 — Core
- Wallet CRUD (nama, tipe bebas, icon, warna, saldo awal)
- Entry manual (pemasukan/pengeluaran, wallet, kategori, tanggal, catatan)
- Transfer antar wallet (tercatat sebagai transfer, tidak mempengaruhi cashflow)
- EntriesPage: riwayat dengan filter jenis/kategori, search, grouped by date
- DashboardPage: total saldo, pemasukan/pengeluaran bulanan, cashflow bersih, donut chart distribusi, bar chart analisis
- Default categories seed saat modul diaktifkan

### Phase 2 — AI Scan Struk
- `server/receipt-scanner.ts` — Gemini Vision, prompt ekstrak total/tanggal/items/toko
- `app/api/fintrack/scan-struk/route.ts` — terima multipart image, upload ke Firebase Storage, return auto-fill data
- Mode scan di NewEntryPage — foto/upload → loading → form auto-filled → user review & adjust
- SettingsPage — input API key per-site, fallback ke `process.env.GEMINI_API_KEY`

### Phase 3 — Advanced Features
- AdvancedPage dengan 4 tab:
  - **Budget**: set budget per kategori per periode, tampilkan progress vs aktual
  - **Goals**: target tabungan dengan nama, jumlah target, terkumpul, deadline, progress bar
  - **Hutang & Piutang**: catat hutang/piutang per orang, tandai lunas
  - **Recurring**: daftar transaksi berulang, frekuensi, tanggal berikutnya, aktif/nonaktif
