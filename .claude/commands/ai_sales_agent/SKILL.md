# /ai_sales_agent — AI Sales Agent Module

You are working on the **Clicker Platform AI Sales Agent Module**. This system integrates Google's Gemini API to provide tenants with a customizable chatbot that answers customer queries and attempts to capture lead information.

This skill is invoked as `/ai_sales_agent [action]`.

---

## 1. Architecture Overview

- **Module Directory:** `lib/modules/ai-sales-agent` (Note the hyphens).
- **Server-Side Integration:** Unlike many frontend modules, the AI Agent heavily relies on a secure Node.js backend to protect Google API keys.
  - `server/gemini-client.ts` houses the actual `@google/genai` SDK implementation.
  - The client UI communicates with Next.js API Routes (e.g., `/api/ai/chat`), which in turn call the `gemini-client.ts`.

---

## 2. Configuration & Prompts

Tenants can customize the AI's behavior via the `AISalesAgentConfig` object (stored in Firestore at `sites/{siteId}/modules/ai-sales-agent/settings/config`).

- `enabled`: Global toggle for the chatbot.
- `systemPrompt`: The core behavioral instruction (e.g., "You are a helpful assistant for Bob's Burgers. Be cheerful.").
- `businessContext`: Additional situational data injected into the prompt (e.g., "We are currently out of stock on fries. We close at 9pm.").
- `greetingMessage`: The first message the user sees.

### Prompt Assembly Rule

When the frontend sends a chat request to the API route, the server must:

1. Fetch the tenant's `AISalesAgentConfig` from Firestore.
2. Concatenate the `systemPrompt` and `businessContext`.
3. Provide this concatenated string as the `systemInstruction` configuration when initializing the Gemini model session.

---

## 3. Lead Capture Flow

The ultimate goal of the AI Sales Agent is to capture leads (`SalesLead` objects).

- If the AI successfully convinces a user to provide their email or phone number in the chat, the system should extract that information.
- This is typically handled by instructing the LLM (via the system prompt) to output a specific JSON structure when a lead is captured, or by using **Function Calling** (Tools) in the Gemini API.
- Once extracted on the server, a new `SalesLead` doc is written to the tenant's database, or optionally, automatically pushed into the **Sales Pipeline** module if integrated.

## Common Gotchas

- **Never Expose API Keys:** The Gemini API Key MUST be stored in a `.env` file (`GEMINI_API_KEY`) and only accessed on the server. Never import `@google/genai` in a Client Component.
- **Streaming UI:** For a natural UX, the chat response from the API route should utilize streaming (`Response.body` readable streams) so words appear as they are generated by Gemini.
- **Token Limits & History:** When sending `ChatMessage` history to the API, be mindless of context windows. You may need to truncate older messages if the conversation gets too long.
